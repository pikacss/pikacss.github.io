<task type="auto">
  <name>Task 2: Implement markdown parser with context detection</name>
  <files>
    packages/api-verifier/src/parser.ts
  </files>
  <action>
Implement documentation parser following RESEARCH.md patterns:

1. Create parser.ts with core functions:
   - `getDocumentationType(filePath: string): DocumentationType`:
     * Returns API_REFERENCE for files matching **/advanced/api-reference.md
     * Returns GUIDE for files in **/guide/** or **/llm/**
     * Returns EXAMPLE for files in **/examples/**
     * Returns OTHER as default
   
   - `normalizeSignature(signature: string): string`:
     * Remove extra whitespace: .replace(/\s+/g, ' ')
     * Normalize colons, arrows, unions: .replace(/\s*:\s*/g, ': ')
     * Trim leading/trailing whitespace
     * Consistent formatting for comparison
   
   - `parseDocumentedAPIs(markdownFile: string): DocumentedAPI[]`:
     * Read file content with fs.readFileSync()
     * Parse with unified().use(remarkParse).parse(content)
     * Use visit(tree, 'code', callback) to find TypeScript code blocks
     * Filter for code blocks with lang: 'typescript' or 'ts'
     * Extract API signatures from code blocks:
       - Match function declarations: /^export (function|const) (\w+)/
       - Match interface declarations: /^export interface (\w+)/
       - Match type declarations: /^export type (\w+)/
       - Match class declarations: /^export class (\w+)/
     * Skip code blocks containing "// example" or "Example:" (user code, not API signatures)
     * Create DocumentedAPI for each match with file, line (from node.position), signature, context

2. Signature extraction heuristics:
   - For functions: extract full signature including params and return type
   - For interfaces: extract interface name + key properties
   - For types: extract type alias name + definition
   - Handle multi-line signatures by joining lines

3. Error handling:
   - Wrap file read in try-catch, return empty array on errors
   - Log warnings for unparseable code blocks
   - Continue processing remaining blocks if one fails

4. Follow RESEARCH.md "Pattern 3: Markdown Code Block Extraction" and "Pattern 5: Context-Aware Documentation Validation"
