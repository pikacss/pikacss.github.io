  <action>
Implement API extraction using TypeScript Compiler API following RESEARCH.md patterns:

1. Create extractor.ts with core functions:
   - `getMonorepoPackages()`: Discovers all 8 packages by reading pnpm-workspace.yaml and filtering for @pikacss/* packages
   - `getPackageEntryPoints(packagePath: string)`: Reads package.json exports field, returns .d.ts file paths (prefer .d.mts for ESM)
   - `extractPackageAPIs(entryPoint: string)`: Uses ts.createProgram() and ts.TypeChecker to extract all exported symbols with full signatures

2. TypeScript extraction logic:
   - Create program with moduleResolution: NodeNext, target: ESNext
   - Use checker.getExportsOfModule() to get all public exports
   - For each export symbol, extract:
     * name: symbol.getName()
     * kind: function|class|interface|type|variable (via ts.isXxxDeclaration checks)
     * signature: checker.typeToString() with NoTruncation flag
     * parameters: for functions, extract param names, types, optional flags
     * returnType: for functions, extract return type string
     * typeParameters: for generic types, extract constraint information
     * deprecated: check JSDoc tags for @deprecated
   - Handle re-exports and type aliases
   - Preserve generic constraints using TypeFormatFlags.WriteTypeArgumentsOfSignature

3. Error handling:
   - Wrap in try-catch, return errors in APIExtractionResult.errors array
   - Handle missing .d.ts files (package may not be built yet)
   - Log warnings for unresolved symbols but continue extraction

4. Add typescript to devDependencies if not already present

5. Follow patterns from RESEARCH.md "Extracting All Public Exports from Package" example
  </action>
