Replace `"typescript": "catalog:"` with `"typescript": "^5.9.3"` in all three fixture package.json files.

**Root cause:** Integration tests copy fixtures to isolated temp directories via `mkdtemp()`. The pnpm catalog protocol (`catalog:`) requires `pnpm-workspace.yaml` with catalog configuration to resolve versions. Isolated environments don't have workspace context, causing ERR_PNPM_CATALOG_ENTRY_NOT_FOUND_FOR_SPEC errors.

**Fix:** Use explicit version matching workspace catalog entry (`typescript: ^5.9.3` from pnpm-workspace.yaml catalog section). This maintains version consistency with monorepo while enabling isolated execution.

**Why not copy pnpm-workspace.yaml:** Would break test isolation principle — fixtures should be self-contained and not depend on parent workspace structure.
