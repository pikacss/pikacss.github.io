<task type="auto">
  <name>Create file reference validator</name>
  <files>scripts/validate-file-refs.js, package.json</files>
  <action>
Create `scripts/validate-file-refs.js` that:

1. **Scans all markdown files** (excluding node_modules, .planning)
2. **Extracts file:line patterns** using RegExp:
   - Pattern: `(\w+(?:\/[\w-]+)*\.(ts|js|vue|mjs|json)):(\d+)`
   - Captures: filepath and line number
3. **Validates references:**
   - Check file exists in workspace
   - Check line number is within file bounds
   - Report missing files or out-of-range line numbers
4. **Output format:**
   ```
   ✓ Valid: packages/core/src/engine.ts:42
   ✗ Invalid: packages/core/src/missing.ts:10 (file not found)
   ✗ Invalid: packages/core/src/engine.ts:999 (line out of range, max: 500)
   ```

Implementation approach:
- Use Node.js fs/promises for file operations
- Use glob pattern to find markdown files
- Store results in JSON format for CI parsing
- Exit code 0 if all refs valid, 1 if any invalid

Add script to package.json:
```json
{
  "scripts": {
    "validate:file-refs": "node scripts/validate-file-refs.js",
    "validate:file-refs:json": "node scripts/validate-file-refs.js --json > file-refs-report.json"
  }
}
```

Handle edge cases:
- Relative paths (e.g., `./src/file.ts:10`) - resolve from markdown file location
- Package-scoped paths (e.g., `@pikacss/core/src/engine.ts:10`) - resolve from monorepo root
- Code blocks containing file:line that aren't meant as references - use heuristics (e.g., ignore if in code fence)
  </action>
  <verify>
Test validator on known file references:

```bash

echo "Check packages/core/README.md:1" > test-refs.md

# Should fail (non-existent file)
echo "Check packages/fake/missing.ts:10" >> test-refs.md

pnpm validate:file-refs

# Clean up
rm test-refs.md
```

Verify output distinguishes valid from invalid references.
  </verify>
  <done>
- scripts/validate-file-refs.js created with file existence and line range validation
- Script handles relative paths, package-scoped paths, and edge cases
- package.json includes validate:file-refs scripts
- Validator tested on sample references
  </done>

<task type="auto">
  <name>Create placeholder detection script</name>
  <files>scripts/detect-placeholders.js, package.json</files>
  <action>
Create `scripts/detect-placeholders.js` that:

1. **Scans all markdown files** (excluding node_modules, .planning)
2. **Detects placeholder patterns:**
   - TODO markers: `TODO`, `todo`, `@todo`
   - FIXME markers: `FIXME`, `fixme`, `@fixme`
   - TBD markers: `TBD`, `tbd`
   - Explicit placeholders: `[placeholder]`, `[TBD]`, `[TODO]`
   - Ellipsis-only content: Lines with only `...` or `…`
   - Stub sections: Headings followed immediately by empty content or "Coming soon"
3. **Output format:**
   ```
   docs/guide/basics.md:42: TODO: Add example for custom plugins
   docs/advanced/api-reference.md:105: [placeholder] - Complete API signature
   packages/core/README.md:20: FIXME: Update build-time constraint explanation
   ```
4. **Summary statistics:**
   ```
   Total placeholders found: 47
   - TODO markers: 23
   - FIXME markers: 8
   - TBD markers: 5
   - Explicit placeholders: 11
   ```

Implementation:
- Use RegExp with line number tracking
- Categorize by placeholder type
- JSON output option for CI integration
- Exit code 1 if any placeholders found (treat as blocker)

Add scripts to package.json:
```json
{
  "scripts": {
    "validate:placeholders": "node scripts/detect-placeholders.js",
    "validate:placeholders:json": "node scripts/detect-placeholders.js --json > placeholders-report.json"
  }
}
```

Exclude false positives:
- Code blocks showing example TODO comments (use heuristics: ignore in code fences)
- Git commit message examples
- Historical content describing removed placeholders
  </action>
  <verify>

Verify all placeholder types detected with correct line numbers.
