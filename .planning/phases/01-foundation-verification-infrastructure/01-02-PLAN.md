<task type="auto">
  <name>Create internal link validation script</name>
  <files>scripts/check-links.sh</files>
  <action>
Create `scripts/check-links.sh` that validates INTERNAL markdown links only (no external URLs to avoid rate limits):

1. Find all markdown files (excluding .planning/**)
2. Extract internal links: `[text](path)`, `[text](path#anchor)`
3. For each link:
   - Resolve relative paths from link's source file
   - Check target file exists
   - If anchor exists, validate heading exists in target file (convert heading to anchor format)
4. Report broken links with file:line context

Exit with non-zero code if any broken links found.

IMPORTANT:
- Only check internal links (relative paths, not http://)
- Handle both file links (`./file.md`) and anchor links (`#heading`)
- Convert markdown headings to anchor format (lowercase, spaces to dashes)
- Exclude .planning/** directory
- Bash script is fine per user guidance (no cross-platform concerns)

Example output format:
```
docs/guide/basics.md:15: Broken link to './nonexistent.md'
docs/advanced/api.md:42: Broken anchor link to './core.md#invalid-section'
```

<task type="auto">
  <name>Create file reference validator</name>
  <files>scripts/check-file-refs.sh</files>
  <action>
Create `scripts/check-file-refs.sh` that validates file:line code references in markdown:

1. Find all markdown files (excluding .planning/**)
2. Extract file:line patterns using regex: `([a-zA-Z0-9_/.@-]+):(\d+)` or similar
3. Common patterns in docs:
   - Inline code: `file.ts:42`
   - Code blocks with file refs
   - @file references in VitePress
4. For each reference:
   - Check file exists (resolve relative to project root or docs root)
   - Check line number doesn't exceed file length
5. Report invalid references with context

Exit with non-zero code if any invalid references found.

IMPORTANT:
- Match realistic file:line patterns (avoid false positives from URLs like http://example.com:8080)
- Handle both absolute paths and relative paths
- Support @/ alias (resolves to docs/ or project root)
- Exclude .planning/** directory

Example output format:
```
docs/advanced/testing.md:78: Invalid reference 'src/nonexistent.ts:10' (file not found)
packages/core/README.md:23: Invalid reference 'src/engine.ts:999' (line 999 exceeds file length 150)
```

2. Add intentional invalid reference to test file
3. Verify script detects invalid reference
4. Add reference to valid file with invalid line number

  </verify>
  <done>
