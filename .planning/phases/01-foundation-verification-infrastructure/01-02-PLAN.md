<task type="auto">
  <name>Set up markdown-link-check for link validation</name>
  <files>package.json</files>
  <action>
Install markdown-link-check for validating internal and external links:
```bash
pnpm add -D -w markdown-link-check
```

Add validation scripts to root package.json:
```json
{
  "scripts": {
    "check-links": "markdown-link-check --config .markdown-link-check.json --quiet $(find . -name '*.md' ! -path './node_modules/*' ! -path './.planning/*')",
    "check-links:verbose": "markdown-link-check --config .markdown-link-check.json $(find . -name '*.md' ! -path './node_modules/*' ! -path './.planning/*')"
  }
}
```

Create `.markdown-link-check.json` configuration:
```json
{
  "ignorePatterns": [
    {
      "pattern": "^https://github.com/.*/pull/"
    },
    {
      "pattern": "^https://github.com/.*/issues/"
    }
  ],
  "timeout": "10s",
  "retryOn429": true,
  "retryCount": 3,
  "aliveStatusCodes": [200, 206, 301, 302, 304]
}
```

Rationale:
- Ignore PR/issue links (often private or ephemeral)
- Retry on rate limiting (429 responses common for external APIs)
- Accept redirect status codes (301, 302) as valid
- Timeout prevents hanging on slow external sites
- Exclude node_modules and .planning directories
  </action>
  <verify>
Run link check on a small subset to test configuration:
```bash
pnpm check-links:verbose README.md
```

Should report:
- Internal links validated against file existence
- External links checked with HTTP requests
- Clear output showing passed/failed links
  </verify>

<task type="auto">
  <name>Integrate link validation into CI pipeline</name>
  <files>.github/workflows/ci.yml</files>
  <action>
Add link validation job to existing CI workflow at `.github/workflows/ci.yml`:

If no ci.yml exists, create one. If it exists, add a new job:

```yaml
name: CI

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main, develop]

jobs:
  # ... existing jobs ...
  
  link-validation:
    name: Validate Documentation Links
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Setup pnpm
        uses: pnpm/action-setup@v2
        with:
          version: 8
      
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'pnpm'
      
      - name: Install dependencies
        run: pnpm install --frozen-lockfile
      
      - name: Check documentation links
        run: pnpm check-links
```

Job runs on every push and PR, failing the build if broken links detected.

Note: If ci.yml already has comprehensive steps, integrate link checking as an additional step in an existing job rather than creating a separate job (to save CI minutes).
