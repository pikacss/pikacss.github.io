# Phase 11: Rules Engine (Implementation)

This plan implements the core `pika-build-time` ESLint rule, which enforces static analysis constraints on `pika()` calls.

## Goals

1.  Add necessary dev dependencies for testing rules (`@typescript-eslint/parser`).
2.  Implement `pika-build-time` rule logic to strictly forbid dynamic arguments.
3.  Export the rule from the package entry point.
4.  Verify the rule with comprehensive unit tests using `RuleTester`.

## Tasks

### 1. Setup Dependencies
Add `@typescript-eslint/parser` as a dev dependency to `packages/eslint-config`. This is required for `RuleTester` to parse TypeScript code in tests.

```bash
# In packages/eslint-config
pnpm add -D @typescript-eslint/parser
```

### 2. Implement Rule Logic
Create `packages/eslint-config/src/rules/pika-build-time.ts`.
- **Target**: `CallExpression`.
- **Logic**:
  - Check if callee name matches configured list (default: `['pika', 'pikap']`) or member expression (e.g., `pika.str`).
  - Iterate through arguments.
  - Recursively check each argument node.
  - **Allowed**: `Literal`, `ObjectExpression` (if values are static), `ArrayExpression` (if elements are static), `TemplateLiteral` (if no expressions).
  - **Disallowed**: `Identifier`, `BinaryExpression`, `CallExpression`, `ConditionalExpression`, `SpreadElement`, etc.
- **Report**: Error on the specific dynamic node with message: "Argument must be statically evaluable (no variables or expressions)."

### 3. Export Rule
Update `packages/eslint-config/src/index.ts`.
- Export a plugin object conforming to ESLint plugin structure.
- `rules`: `{ 'pika-build-time': pikaBuildTime }`.
- `meta`: Plugin metadata.

### 4. Create Tests
Create `packages/eslint-config/tests/rules/pika-build-time.test.ts`.
- Use `RuleTester` from `@typescript-eslint/rule-tester` (or `eslint` if compatible with Vitest globals).
- **Test Cases**:
  - Valid: `pika({ color: 'red' })`, `pika.str('foo')`, `pika(['a', 'b'])`, `pika({ nested: { val: 1 } })`.
  - Invalid: `pika(someVar)`, `pika({ color: someVar })`, `pika('a' + 'b')`, `pika(\`val-${x}\`)`, `pika(condition ? 'a' : 'b')`.

### 5. Verify Build & Test
- Run `pnpm build` in `packages/eslint-config` to ensure clean build.
- Run `pnpm test` to execute the new tests.

## Verification

- **Must Have**:
  - `pika-build-time` rule correctly identifies dynamic arguments.
  - Unit tests pass for all valid/invalid cases.
  - Build artifacts include the rule definition.
