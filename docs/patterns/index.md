# Patterns

The pattern pages cover repeatable application architecture and team practices for PikaCSS projects. Each page focuses on one recurring problem and how to solve it with the PikaCSS model.

## Which pattern to read first

Use this decision guide:

| Your situation | Read this first |
| --- | --- |
| Building a single reusable component | [Component Styling](/patterns/component-styling) |
| Adding hover states, focus rings, or responsive breakpoints | [Responsive And Selectors](/patterns/responsive-and-selectors) |
| Storing per-instance dynamic values at runtime | [Dynamic Values With CSS Variables](/patterns/dynamic-values-with-css-variables) |
| Building a design system with shared tokens | [Theming And Variables](/patterns/theming-and-variables) |

Start with [Component Styling](/patterns/component-styling) if you are not sure. It establishes the base pattern that the other pages extend.

## Pattern pages

- [Component Styling](/patterns/component-styling) — structure a PikaCSS component with base styles, variants, and state layers.
- [Responsive And Selectors](/patterns/responsive-and-selectors) — apply selector aliases for hover, focus, dark mode, and breakpoints.
- [Dynamic Values With CSS Variables](/patterns/dynamic-values-with-css-variables) — bridge static extraction with runtime-varying values using CSS custom properties.
- [Theming And Variables](/patterns/theming-and-variables) — build a design token layer and apply it across components.
